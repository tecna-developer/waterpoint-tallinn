# WaterPoint Tallinn

PWA-прототип приложения для поиска общественных питьевых кранов в Таллине.
Дизайн — по макетам Stitch «WaterPoint Tallinn» (светлая тема, финальная итерация),
функциональность — по ТЗ v1.0 (15.07.2026).

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # сборка в dist/
```

| FR-02 Геолокация | ✅ запрос разрешения, сортировка по расстоянию; при отказе — баннер и ручной поиск |
| FR-03 Поиск | ✅ районы (linnaosa), адреса точек, онлайн-геокодинг Nominatim (ограничен Таллином), подсказка при невалидном запросе |
| FR-04 Карта + список | ✅ одинаковый набор результатов; маркеры различают статусы цветом **и** глифом |
| FR-05 Карточка | ✅ название, район/асум, расстояние, тип, сезон, статус, код VVK, описание, Route/Favorite/Report |

| FR-07 Фильтры | ✅ доступно сейчас, ≤1/2/5 км, outdoor/indoor, для животных, избранное; сброс; счётчик результатов |
| FR-08 Избранное + офлайн | ✅ localStorage без аккаунта; офлайн-баннер с датой кеша; service worker кеширует оболочку |
| FR-09 Сообщение о проблеме | ✅ 5 категорий, комментарий, контакт по желанию, согласие, honeypot + rate-limit, подтверждение |

Статусы точек — по §5 ТЗ: `available` («доступна по последним данным», без live-гарантии),
`seasonal_closed` (вне окна май–октябрь), `reported_issue` (предупреждение до модерации,
официальный статус не меняется), `temporarily_unavailable`, `unknown`.

## Данные

- Источник: Tallinn GIS ArcGIS REST — слой «Avalikud veevõtukohad».
  Снапшот в `src/data/waterpoints.json`; районы вычислены по полигонам `Linnaosad_asumid`.
- Фото точек в источнике нет — используются нейтральные заглушки (без выдуманного контента).
- Перед релизом подтвердить условия повторного использования данных (этап 0 ТЗ).

## Приватность

Точная геолокация не сохраняется и никуда не отправляется; избранное, кеш и отчёты — только на устройстве.
